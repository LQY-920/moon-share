## 图 3 · 四个会话的分工

**标题**：
我的电脑上同时跑着 4 个 AI

🟦 左上｜内容分享
→ 把开发经验沉淀成公众号稿

🟨 右上｜仓库管理
→ review spec + 提交 git

🟩 左下｜dev
→ subagent 跑 21 个 Task 写代码

🟥 右下｜plan
→ 规划下一步执行策略

**对内 3 个会话做产品**
**对外 1 个会话做内容**

---

## 图 4 · 核心原则 ①

**标题**：
① 按"心智模式"切分

| 会话 | 心智模式 |
|---|---|
| 内容分享 | 讲述者 |
| 仓库管理 | 审查者 |
| dev | 工匠 |
| plan | 指挥官 |

**四种心智模式不能混在一个会话里**
混在一起，AI 会在"创作"和"审查"之间切换
上下文被污染

---

## 图 5 · 核心原则 ②

**标题**：
② 人是编排者，不是执行者

❌ 我不写代码
❌ 我不写文章  
❌ 我不 review 每一行

✅ 我在**关键决策点**出现：
- 选题方向
- 架构拍板
- 是否放行

**我的瓶颈不是手速
是决策速度**

---

## 图 6 · 核心原则 ③

**标题**：
③ 通过文件协同，不是对话

plan 写 plan.md
dev 读 plan.md 开工
dev 写代码
review 读代码 review

**任何会话崩溃都不影响其他**
异步协同 = 鲁棒性

---

## 图 7 · 怎么开始

**标题**：
渐进式推进（不用一上来开 4 个）

**阶段 1** · 单会话  
→ 小项目、探索阶段

**阶段 2** · dev + plan  
→ 中等项目，提前规划

**阶段 3** · + review  
→ 需要质量保证

**阶段 4** · + 对外内容  
→ 想打造个人品牌

## 图 8 · 收束

**标题**：
超级个体 = 个人 + AI 团队

```
以前：4 个人 + 协作工具 + 无数会议
现在：1 台电脑 + 4 个 AI 会话
     + 一套清楚的分工规则
```

手速快一倍 → 还是单线
**并行能力增加 → 才是超级个体**
