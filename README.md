# moon-share

AI 实践经验对外分享仓库。面向公众号、朋友圈、视频号、抖音、小红书五个渠道。

## 目录规范

```
moon-share/
├── README.md          # 本文件
├── topics.md          # 选题清单（活文档，随工作推进更新状态）
├── articles/          # 正式稿
│   └── <NN-slug>/     # 每个选题一个目录，数字前缀用于排序
│       ├── gzh.md         # 公众号长文版
│       ├── xhs.md         # 小红书/朋友圈卡片版（多图或图文）
│       ├── script.md      # 视频号 / 抖音口播稿（含分镜）
│       └── assets/        # 本篇配图、截图、流程图
├── drafts/            # 未进入正式稿的草稿、片段
└── refs/              # 引用材料清单（spec 路径、外部文章链接）
```

### 命名约定

- 目录 slug 用中文，数字前缀按立项顺序（`01-四层设计法`）
- 文件名固定三套：`gzh.md` / `xhs.md` / `script.md`，不用就省略
- 配图命名 `01-decision-table.png` 这种可读形式

## 工作流

1. **立项**：在 `topics.md` 添一行，填核心钩子、主打渠道、优先级
2. **草稿**：在 `drafts/` 下写，或直接在 `articles/<slug>/gzh.md` 起头
3. **定稿**：三种渠道版本各自标状态（⬜ → 🟡 → 🟢）
4. **发布**：发完标 🚀，`topics.md` 的"备注"列补发布链接
5. **归档**：不再更新的选题保留目录，只改状态

## 内容原则

- **可以完整引用 moon-agent-os 的 spec 内容**，包括 DDL、代码、具体子系统名
- 方法论先行，项目细节作为例证（不是把 spec 翻译一遍）
- 每篇有一个清晰的 hook，一篇不讲两件事
